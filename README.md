# 🏥 As-Shifa Secure Healthcare Management System

This project is part of the SWE-445: Secure Software Development course at King Fahd University of Petroleum & Minerals (KFUPM).

It is a secure, role-based healthcare management system prototype designed to manage sensitive patient information, appointments, prescriptions, and insurance claims while ensuring data privacy, integrity, and availability according to industry standards like **HIPAA** and **GDPR**.

This repository contains **high-fidelity front-end prototypes (HTML, CSS, JS)** for the main system actors:
- Patients ✅ (Completed)
- Doctor (Coming soon)
- Healthcare Staff (Coming soon)
- System Administrator (Coming soon)

> ⚠️ **Note:** This is **Phase 2a (User Interface Design)** of the project.  
> No backend is connected. All data flows are simulated with mock data and interactions.

---


---

## ✅ Features (Patient Interface)

- **🧩 Modular HTML pages** for key patient functions
- **🔒 Secure login** with multi-factor authentication (MFA)
- **🩺 Medical records** viewer with role-based access control and versioning
- **📤 Secure record sharing** with authorized doctors or facilities
- **💳 Insurance claim submission** with data encryption simulation
- **📊 System admin views** (audit logs & monitoring — reserved for Admin role)
- **🎨 Clean, modern UI** with animations and responsive design
- **🧩 Popups** for profile and help menu, improving navigation usability

---

## 🎮 How to Use

1. Clone or download the repository
2. Open any HTML file in your browser (start with `/html/login.html`)
3. Use the simulated login to navigate between pages
4. Test popups (Profile, Help) and view page animations
5. Review role-based UI (doctors vs. patients vs. admin mockups)

> 💡 **Tip:** No installation is required. Just open in any modern browser (Chrome, Firefox, Edge).

## 🛡️ Security Design Principles Used

- **Authentication & Authorization**: Multi-factor authentication and role-based access control
- **Data Confidentiality**: Simulated data encryption flows
- **Integrity & Logging**: Activity tracking and version history for sensitive data
- **Availability**: DoS monitoring UI prototype
- **Non-repudiation**: Audit logs in tamper-evident design
